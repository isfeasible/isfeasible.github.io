空格引发的段、图表等等之类解析错误

在不同段之间一定要用空格分开，不然有可能导致解析错误。比如图表行直接与文字行紧挨着，图表将不能被正常解析，而在两行之间加入一个空行图表即可正常解析。

Markdown 空格、换行的解决方法

因为Markdown文件最终会被解析成html，所以只要参考html中空格、换行的写法即可。在html中”<br />“表示换行，”&nbsp;“表示空格。

在页面显示换行、空格字符的解决方法

这个问题不是在页面中显示空格、换行效果，而是要在页面显示字符”<br />“和”&nbsp;“。

这部分就是特殊字符在html中的显示问题，不同于上面说的链接中特殊字符的显示问题。

空格 & < > “ ‘
&nbsp; &amp &lt; &gt; &quot; &qpos;

所以如果想在页面上显示字符”<br />“，那么Markdown的写法应该是：”&lt;br&nbsp;/&gt;“；空格”&nbsp;“对应的写法应该是：”&amp;nbsp;“。
